## システム概要

本システムは、飲食店（ビスクドールグループ）の朝礼・終礼運用をデジタル化・効率化するためのWebアプリケーションです。Google スプレッドシートと Google Apps Script を活用し、複数店舗の同時運用にも対応します。

### 主な特徴

- **WebApp経由の全操作**: スプレッドシートへの直接アクセス不要（コンフリクト防止）
- **ロール別画面**: 店責用・キャスト用・管理者用の最適化されたUI
- **AM2:00区切り**: 営業日の定義に合わせた柔軟な日付管理
- **スケーラブル**: 8店舗〜50店舗まで対応可能なアーキテクチャ

## 機能一覧

### 店責用機能

| 機能 | 説明 |
|------|------|
| 朝礼入力 | 出勤キャストの登録・目標管理・ニュース共有 |
| 終礼入力 | 売上実績の入力・目標達成判定 |
| 課題起票 | 終礼での課題・要望を登録・トラッキング |

| 機能 | 説明 |
|------|------|
| 目標入力 | 自分の目標を入力・編集 |
| 実績閲覧 | 当日の売上・月間累計を確認（ランキング非表示） |

### 管理者用機能

| 機能 | 説明 |
|------|------|
| 全店舗概況 | 全店舗の当日状況を一覧確認 |
| 課題管理 | 全店舗の課題を一元管理・FB入力 |
| 月間集計 | 店舗別・全体の月間実績集計 |

### その他機能

| 機能 | 説明 |
|------|------|
| 早退者セルフ評価 | Google Form連携で早退者の振り返りを収集 |
| LINE通知（オプション） | 未対応課題が7日以上経過で自動通知 |

## デプロイ手順

### Step 1: Google スプレッドシートの作成

1. Google スプレッドシートを新規作成
2. 名前を「朝礼・終礼シート」等に設定

### Step 2: Google Apps Script プロジェクトの作成

1. スプレッドシートから「拡張機能」→「Apps Script」を開く
2. プロジェクト名を設定（例: 朝礼終礼システム）

### Step 3: コードの配置

以下のファイルを作成・配置:

1. `Code.gs` - メイン処理・初期化
2. `API.gs` - API関数（認証・データ操作）
3. `Form.gs` - Google Form連携
4. `appsscript.json` - マニフェストファイル
5. `index.html` - メインHTML
6. `CSS.html` - スタイルシート
7. `JavaScript.html` - クライアント側スクリプト

### Step 4: スプレッドシートの初期化

1. Apps Script エディタで `Code.gs` を開く
2. 関数選択から `setupSpreadsheet` を選択
3. 「実行」ボタンをクリック
4. 権限のリクエストが表示された場合は許可
5. シートが自動生成されることを確認

### Step 5: 店舗マスタの設定

1. 「マスタ」シートを開く
2. サンプルデータを削除または編集
3. 実際の店舗情報を入力:

| 店舗名 | ブランド | エリア | 店責パスワードハッシュ | 管理者パスワードハッシュ |
|--------|---------|--------|------------------------|-------------------------|
| 東京店 | ビスクドール | 関東 | | |
| 大阪店 | ビスクドール | 関西 | | |

**注意**: パスワードハッシュは後の手順で設定します。

### Step 6: パスワードハッシュの生成

Apps Script エディタで以下を実行:

```javascript
// 店責パスワードのハッシュを生成（例: password123）
function generateManagerPassword() {
  console.log(getHash('password123'));
}

// 管理者パスワードのハッシュを生成（例: admin123）
function generateAdminPassword() {
  console.log(getHash('admin123'));
}
```

ログに出力されたハッシュ値を「マスタ」シートの該当列に貼り付け。

### Step 7: Google Formの作成（オプション）

1. Apps Script エディタで `Form.gs` を開く
2. 関数選択から `createEarlyLeaveForm` を選択
3. 「実行」ボタンをクリック
4. 生成されたフォームURLが「マスタ」シートのF2セルに保存される
5. `generateQRCode` を実行してQRコードを印刷

### Step 8: WebApp のデプロイ

1. 「デプロイ」→「新しいデプロイ」を選択
2. 種類を「ウェブアプリ」に選択
3. 以下の設定でデプロイ:

| 項目 | 設定値 |
|------|--------|
| 説明 | 朝礼・終礼システム |
| 実行ユーザー | 自分（デプロイ者） |
| アクセスできるユーザー | 全員 |

4. 「デプロイ」ボタンをクリック
5. WebアプリのURLが発行される

### Step 9: 動作確認

1. デプロイされたURLにアクセス
2. 以下のアカウントでログインできることを確認:

| ロール | 店舗 | パスワード（例） |
|--------|------|------------------|
| 店責 | 東京店 | password123 |
| キャスト | 東京店 | password123 |
| 管理者 | - | admin123 |

## 使い方

### 店責の方へ

#### 朝礼入力

1. WebAppにログイン（店責モード）
2. 「朝礼入力」タブを開く
3. 出勤キャストを追加（「+ キャスト追加」ボタン）
4. 各キャストの名前・契約時間・送迎の有無を入力
5. 店舗ニュース・個人ニュースがあれば入力
6. 「保存」ボタンをクリック

#### 終礼入力

1. 「終礼入力」タブを開く
2. 朝礼で入力したキャスト一覧が自動表示
3. Airegiを確認しながら各キャストの実績を入力
4. 店舗全体売上を入力
5. 「保存」ボタンをクリック

#### 課題の追加

1. 終礼入力画面の下部「課題・要望を追加」
2. 内容と起票者を入力
3. 「課題を追加」ボタンをクリック

### キャストの方へ

#### 目標入力

1. WebAppにログイン（キャストモード）
2. 店舗を選択
3. 名前と本日の目標を入力
4. 「保存」ボタンをクリック

#### 実績閲覧

1. 「実績閲覧」タブを開く
2. 名前を入力
3. 「実績を表示」ボタンをクリック
4. 当日の売上・月間累計が表示される

### 管理者の方へ

#### 全店舗概況

- 全店舗のキャスト数・売上を一覧確認
- リアルタイムの営業状況を把握

#### 課題管理

- ステータス別に課題をフィルタリング
- FBコメントを入力して店責経由で返信

#### 月間集計

- 店舗別または全体の月間実績を集計
- 目標達成率の確認

## トラブルシューティング

### ログインできない

- パスワードハッシュが正しく設定されているか確認
- 「マスタ」シートの店舗名と選択した店舗名が一致しているか確認

### データが保存されない

- スプレッドシートの権限を確認（デプロイ者が編集権限を持っているか）
- WebAppのデプロイ設定を確認（「実行ユーザー: 自分（デプロイ者）」）

### 朝礼/終礼のシートが自動作成されない

- 月が変わった際に最初のアクセスで自動作成されます
- 手動で作成する場合は、Apps Script エディタで `createChoreiSheet()` または `createShureiSheet()` を実行

### 日付がおかしい

- AM2:00区切りのロジックが適用されています
- 営業日の定義に合わせて `DATE_CUTOVER_HOUR` の値を調整してください（`Code.gs`内）

### WebAppが開かない

- アクセス権限が「全員」になっているか確認
- 再デプロイを試してください

## セキュリティ上の注意

1. **パスワード管理**
   - 店責が退職した場合はパスワードを変更してください
   - 管理者パスワードは定期的に変更することを推奨

2. **データの保護**
   - スプレッドシートへの直接アクセスは本部（管理者）のみに制限
   - キャストには当日と累計の実績のみ表示（過去の日別データは非表示）

3. **URLの管理**
   - WebAppのURLは社内でのみ共有してください
   - URLが漏れた場合はパスワード変更またはWebAppの再デプロイを検討

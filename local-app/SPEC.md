# 朝礼・終礼シートシステム 仕様書 v2

## 1. 概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | 朝礼・終礼シートシステム |
| 企業 | 株式会社esplanade |
| 開発フェーズ | ローカル版（Node.js） → 最終形態：GAS Web App |
| データ保存先 | ローカル版: SQLite / GAS版: Google スプレッドシート |
| 認証 | Googleアカウント（全ロール共通） |
| 想定同時アクセス | 最大50程度 |

---

## 2. ユーザーロール

| ロール | 説明 | 認証 |
|--------|------|------|
| 店責 (manager) | 店舗責任者。朝礼・終礼の入力・管理 | Googleアカウント |
| キャスト (cast) | スタッフ。目標・送迎・振り返りの入力、閲覧 | Googleアカウント（キャスト名はマスタで紐付け） |
| 専任 (admin) | 本部。全店舗の閲覧・集計・伝言板ステータス管理 | Googleアカウント |

### キャスト識別

- イントラのGoogleスプレッドシートに `Gmail | キャスト名` の対応表がある
- システムはこのスプシを参照し、Googleアカウントからキャスト名を自動解決
- キャストは複数店舗かけもちあり（その日どの店舗で働くかは、店責が朝礼で追加して決まる）

---

## 3. 営業日ロジック

- **AM 11:00 区切り**: 3/15の営業日 = 3/15 11:00 〜 3/16 10:59
- 朝礼: 17:50頃
- 終礼: 26:00（翌2:00）以降、遅くなる可能性あり
- フォーマット: `YYYY-MM-DD`

## 4. データモデル

| カラム | 型 | 説明 |

### 4.5 shurei（終礼データ）

| カラム | 型 | 説明 |

### 4.7 issues（伝言板）

| カラム | 型 | 説明 |

| id | INTEGER PK | |

## 5. 画面構成

### 5.1 ログイン画面

- Googleアカウントでログイン
- ロールはアカウントに紐付いているため自動判定
- キャスト・店責の場合 → 店舗選択（かけもち対応）
- ローカル版ではロール選択 + 店舗選択で代替

### 5.2 店責用画面

#### タブ①: 朝礼

**入力項目（キャストごと）:**
| 項目 | 入力者 | 説明 |
|------|--------|------|
| キャスト名 | 店責 | マスタから選択 or 手入力で追加 |
| 当月累計売上 | 店責 | 手入力 |
| 当月累計ドリンク数 | 店責 | 手入力 |
| メモ | 店責 | 自由記述 |
| 目標 | キャスト | キャスト自身が入力 |
| 送迎 | キャスト | 要否 + 場所 |

**操作:**
- キャスト追加・削除
- 保存

#### タブ②: 終礼

**入力項目:**
| 項目 | 説明 |
|------|------|
| 現金売上 | 円 |
| カード売上 | 円 |
| PayPay売上 | 円 |
| RoseLink売上 | 円 |
| 合計 | 自動計算（リアルタイム表示） |
| 今月の店舗売上 | 手入力 |

**操作:**
- 保存

#### タブ③: 伝言板

- 日付ごとの投稿一覧（時系列）
- ステータスタブ: 未対応 / 対応中 / 完了
- 新規投稿フォーム: 内容 + 投稿者名
- 各投稿にステータスバッジ表示

#### タブ④: 自己採点一覧

- 当日の全キャストの採点・振り返り一覧
- 誰が書いて誰がまだか分かる

### 5.3 キャスト用画面

#### タブ①: 入力（目標・送迎）

- 自分の目標入力（テキストエリア）
- 送迎: 要否トグル + 場所入力
- 保存ボタン
- ※ キャスト名はGoogleアカウントから自動解決されるため名前入力不要

#### タブ②: 朝礼（閲覧）

- 今日の出勤キャスト一覧テーブル
- 名前、当月累計売上、当月累計ドリンク数、目標

#### タブ③: 終礼（閲覧）

- 店舗合計売上（4要素 + 合計）
- 今月の店舗売上

#### タブ④: 振り返り

- 10段階自己採点ボタン
- 振り返りコメント（テキストエリア）
- 早退フラグチェックボックス
- 保存ボタン
- ※ 全キャストが営業終了後に書く。早退者は帰る前に書く

#### タブ⑤: 伝言板

- 投稿一覧（閲覧）
- 新規投稿も可能

### 5.4 本部画面（専任）

#### タブ①: 全店ダッシュボード

- 全店舗テーブル: 店舗名、出勤キャスト数、当日売上合計
- 日付選択で過去分も閲覧可能

#### タブ②: 伝言板（全店）

- 全店舗の伝言を一覧
- **ステータス変更UI**: 未対応→対応中→完了 に変更可能
- フィードバックコメント入力
- ステータスフィルタ

#### タブ③: 月間集計

- 店舗選択（全店 or 個別）
- 月間売上合計
- 今月の店舗売上推移

#### タブ④: 過去データ編集

- 日付 + 店舗を選択して過去の朝礼・終礼データを編集
- 専任（バックヤード）のみ操作可能

## 6. データの流れ

### 朝礼フロー（17:50頃）
```
店責: キャスト追加（マスタから選択）
      ↓
      各キャストの当月累計売上・ドリンク数を入力
      ↓
      メモを入力
      ↓
      保存
      ↓
キャスト: 自分の画面で目標を入力 → 保存
キャスト: 送迎情報を入力 → 保存
      ↓
朝礼実施（画面を見ながら）
```

### 終礼フロー（26:00頃〜）
```
店責: 終礼タブを開く
      ↓
      売上4要素（現金・カード・PayPay・RoseLink）を入力
      合計は自動計算
      今月の店舗売上を入力
      ↓
      保存
      ↓
キャスト: 振り返りタブで自己採点（10段階）+ コメントを入力 → 保存
      ↓
早退キャスト: 帰る前に振り返りを書く
      ↓
店責: 自己採点一覧タブで全員の振り返りを確認
      ↓
伝言板: 現場からの意見・要望を投稿
      ↓
専任: 伝言板を確認 → ステータス更新・フィードバック
```

### 認証フロー（GAS版）
```

  ↓

  ↓

ロール管理スプシ/シートでロール判定
  ↓
ロールに応じた画面を表示
```

## 8. ローカル版での割り切り

GAS最終形態に向けたローカル開発版では以下を簡略化する:

| 項目 | ローカル版 | GAS版 |
|------|-----------|-------|
| 認証 | ロール選択 + 店舗選択（パスワードなし） | Googleアカウント自動判定 |
| キャスト識別 | 名前手入力 | Gmail → キャスト名自動解決 |
| データ保存 | SQLite | Googleスプレッドシート |
| 営業日 | 11:00区切りで動的計算 | 同左 |
| 過去データ編集 | admin画面から可能 | 同左 |
